# قالب واتساب التسويقي + دليل التفعيل الكامل

كل الكود جاهز ومنشور. ما يلي هو **ما يحتاج حساباتك أنت** — لا أستطيع تنفيذه نيابةً عنك لأنه يتطلّب دخولاً إلى Meta.

---

## 2) ربط القالب بالنظام

في **لوحة المالك ← العملاء المحتملين ← «واتساب تسويقي»**:

1. الوضع: **قالب معتمد**
2. اسم القالب: `fieldsales_intro_ar` · اللغة: `ar`
3. المتغيّرات — اضغطها **بهذا الترتيب**:
   - `اسم الشركة` → يصبح `{{1}}`
   - `زاوية الدولة (عربي)` → تصبح `{{2}}`
4. ستظهر **معاينة حيّة** على عميل حقيقي من قاعدتك. تأكّد أن الخانتين ممتلئتان قبل الإرسال.

**لماذا هذا مهم:** `{{2}}` تتغيّر تلقائياً حسب دولة كل عميل من قالب واحد معتمد — السعودي يصله ذكر ZATCA، والمصري ETA، والإماراتي FTA، والعراقي «يعمل مع الإنترنت الضعيف»… **19 دولة، قالب واحد**، بلا 19 مراجعة من Meta.

---

## 3) متغيّرات البيئة على Render (خدمة `dsd-backend`)

| المتغيّر | من أين تجلبه | مطلوب؟ |
|---|---|---|
| `WHATSAPP_TOKEN` | Meta → System User → Generate Token (دائم) | **نعم** |
| `WHATSAPP_PHONE_NUMBER_ID` | WhatsApp Manager → API Setup → Phone number ID | **نعم** |
| `WHATSAPP_VERIFY_TOKEN` | نصّ تختاره أنت (أي شيء عشوائي) | للردود |
| `WHATSAPP_APP_SECRET` | Meta → App → Settings → Basic → App Secret | **موصى به بشدّة** |

> ⚠️ **بخصوص `WHATSAPP_APP_SECRET`:** بدونه يعمل الـwebhook لكن **بلا تحقّق من التوقيع** — أي أن أي طرف يعرف الرابط يستطيع حقن ردود مزيّفة أو انسحابات كاذبة في قاعدة عملائك. ليس عطلاً فورياً، لكنه ثغرة مفتوحة. اضبطه.

---

## 4) ربط الـwebhook (لاستقبال الردود)

في **Meta → App → WhatsApp → Configuration → Webhook**:

- **Callback URL**: `https://api.fieldsa.net/api/whatsapp/webhook`
- **Verify Token**: نفس قيمة `WHATSAPP_VERIFY_TOKEN`
- اضغط **Verify and Save**
- ثم **Manage** واشترك في الحقلين: `messages` و `message_status`

### ماذا يحدث بعدها تلقائياً

| الحدث | الأثر في النظام |
|---|---|
| العميل يردّ | نشاط `WHATSAPP_IN` + المرحلة تُرفع إلى **مؤهَّل** |
| العميل يردّ «إيقاف» / يضغط زرّ الإيقاف | `waOptOut = true` — **يُستثنى من كل إرسال لاحق نهائياً** |
| Meta تُبلّغ التسليم/القراءة | حالة الرسالة تُحدَّث: `DELIVERED` / `READ` |
| فشل الإرسال | `FAILED` + سبب الخطأ، **ولا يستهلك الحصّة اليومية** |

---

## 5) الرقم — ما يجب أن تعرفه

- الرقم **يجب ألّا يكون مسجّلاً على واتساب عادي أو Business App**. شريحتك الجديدة مثالية لهذا.
- إن كان مسجّلاً، احذف الحساب من التطبيق أولاً ثم سجّله في Cloud API.
- **لا تستخدم `+966590633827`** — هو رقمك المنشور على الموقع، وضمّه لـCloud API يعني أنك لن تستقبل عليه واتساب عادياً بعدها.

### الحدود الفعلية

| المستوى | الحدّ |
|---|---|
| قبل توثيق الأعمال | **250** محادثة/24س |
| بعد التوثيق | 1,000 → 10,000 → 100,000 (يرتفع تلقائياً بحسن الأداء) |
| المجاني شهرياً | أول **1,000** محادثة |

الحدّ الافتراضي في النظام **80/يوم** — متحفّظ عمداً. تقييم جودة رقمك في Meta يهبط بالحظر والتجاهل، وهبوطه يقيّد الرقم. **ابدأ منخفضاً وارفع تدريجياً بحسب معدّل الردّ.**

## 6) أسباب الرفض الشائعة

| السبب | العلاج |
|---|---|
| متغيّران متجاوران `{{1}} {{2}}` | ضع نصّاً بينهما (قالبنا سليم) |
| القالب يبدأ أو ينتهي بمتغيّر | قالبنا يبدأ بـ«مرحباً» وينتهي بنصّ ثابت |
| عيّنات فارغة | عبّئ قيم القسم 1 |
| وعود مبالغ فيها | لا تَعِد بأرقام أرباح |
| اسم بأحرف كبيرة أو مسافات | `fieldsales_intro_ar` — صغيرة وشرطات سفلية فقط |

المراجعة تستغرق عادةً **دقائق إلى 24 ساعة**. عند الرفض، عدّل وأعد الإرسال — لا عقوبة على المحاولة.

---

## 7) الترتيب الموصى به

1. فعّل الشريحة الجديدة في Cloud API
2. اضبط `WHATSAPP_TOKEN` + `WHATSAPP_PHONE_NUMBER_ID` → **الإرسال يعمل فوراً**
3. أرسل القالب للاعتماد
4. اضبط الـwebhook + `WHATSAPP_APP_SECRET` → **الردود تعمل**
5. جرّب على **10 أرقام** تعرفها، لا على 80
6. اقرأ معدّل الردّ، ثم ارفع الحدّ

> **لا ترفع الحدّ قبل أن ترى معدّل الردّ.** رسالة لا تعمل × 80 يومياً = رقم مقيَّد خلال أسبوع. رسالة تعمل × 20 يومياً = خطّ عملاء مستدام.
