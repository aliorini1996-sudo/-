# جسر واتساب ويب — Field Sales

يفتح جلسة واتساب ويب حقيقية على **جهازك**، ويرسل الرسائل التي تُدرجها من لوحة المالك.

## التشغيل (مرّة واحدة)

**1) مفتاح مشترك** — اختر نصّاً عشوائياً طويلاً، وضعه في مكانين:
- على Render (خدمة `dsd-backend` ← Environment): `WA_BRIDGE_KEY`
- على جهازك (الخطوة 3)

**2) تثبيت**
```bash
cd wa-bridge
npm install
```
> **يتطلّب Google Chrome مثبّتاً على الجهاز.** الجسر يستخدم متصفّحك الحقيقي —
> لا يُنزّل Chromium خاصاً به. هذا مقصود: متصفّح حقيقي = سلوك أطبع أمام واتساب،
> وبلا 150م.ب إضافية قد ينكسر استخراجها.
>
> يجده تلقائياً في مساراته المعتادة (ويندوز/ماك/لينكس، ويقبل Edge بديلاً).
> إن كان في مكان غير معتاد، حدّده: `WA_BRIDGE_CHROME`.

**3) ضع المفتاح في ملف `.env`**

افتح `wa-bridge\.env` بالمفكّرة واكتب فيه سطراً واحداً:
```
WA_BRIDGE_KEY=المفتاح-من-Render
```
بلا علامات تنصيص وبلا مسافات حول `=`. (القالب في `.env.example`.)

**4) التشغيل**

**ويندوز:** انقر نقراً مزدوجاً على **`تشغيل-الجسر.cmd`** — لا حاجة لأي سطر أوامر.

أو من الطرفية (أي نظام):
```bash
npm start
```

**5) امسح الرمز** — يُطبع **في النافذة نفسها** فور التشغيل. امسحه من هاتفك: واتساب ← الإعدادات ← **الأجهزة المرتبطة** ← ربط جهاز. (يظهر أيضاً في لوحة المالك ← العملاء المحتملين ← «واتساب ويب».)

الرمز يتجدّد كل دقيقة تقريباً — طبيعي. بعد المسح تظهر: `✓ متصل بالرقم …`

## الاستخدام اليومي

انقر **`تشغيل-الجسر.cmd`** نقراً مزدوجاً. أغلق النافذة أو `Ctrl+C` عند الانتهاء.

الجلسة محفوظة في `.session/` — **لا تعيد مسح QR كل مرّة**.

> الجسر يرسل **فقط** والنافذة مفتوحة. أغلقتها ⇒ يتوقّف الإرسال والرسائل تنتظر في الطابور بأمان.

## الإعدادات (اختيارية)

| المتغيّر | الافتراضي | الوظيفة |
|---|---|---|
| `WA_BRIDGE_KEY` | — | **مطلوب** — نفس قيمته على Render |
| `WA_BRIDGE_API` | `https://api.fieldsa.net/api/wa-bridge` | عنوان الخادم |
| `WA_BRIDGE_CHROME` | يُكتشف تلقائياً | مسار Chrome إن لم يُوجد تلقائياً |
| `WA_BRIDGE_POLL_MS` | `8000` | كل كم يسحب الطابور |
| `WA_BRIDGE_MIN_GAP_MS` | `8000` | أقلّ تباعد بين رسالتين |
| `WA_BRIDGE_MAX_GAP_MS` | `25000` | أكبر تباعد بين رسالتين |

**لا تُقلّل التباعد.** الإرسال بإيقاع منتظم سريع هو أوضح إشارة تلتقطها أنظمة واتساب.

## احترس

- **هذا ليس واجهة واتساب الرسمية.** أتمتة واتساب ويب تخالف شروط الاستخدام، وخطر الحظر حقيقي.
- **استخدم شريحة منفصلة** — لا رقمك المنشور على الموقع. الحظر غالباً نهائي.
- **ابدأ بـ10 رسائل يومياً وارفع تدريجياً.** الرقم الجديد بلا تاريخ أضعف ما يكون في أسبوعه الأول.
- **الحظر يأتي من البلاغات لا من العدد.** رسالة سيّئة × 10 أخطر من رسالة جيّدة × 50. حسّن الرسالة قبل أن ترفع الكميّة.
- من ردّ بكلمة «إيقاف» يُستثنى تلقائياً وللأبد — لا تحاول تجاوز ذلك.

## لا تُرفع للمستودع

`.session/` تحوي **توثيق حسابك على واتساب** — من يملكها يملك جلستك. مستثناة في `.gitignore`؛ لا ترفعها ولا تشاركها.

## عطل شائع

| العرَض | السبب |
|---|---|
| `Could not find Chrome` | Chrome غير مثبّت أو في مسار غير معتاد — ثبّته أو حدّد `WA_BRIDGE_CHROME` |
| `HTTP 401` متكرّرة | `WA_BRIDGE_KEY` غير مضبوط على Render، أو مختلف عن قيمته لديك |
| رمز QR يتجدّد كل دقيقة | **طبيعي** — واتساب يُجدّده حتى تمسحه |
| `مفتاح جسر غير صالح` | `WA_BRIDGE_KEY` مختلف بين جهازك وRender |
| لا يظهر QR في اللوحة | الجسر لا يصل الخادم — تحقّق من الإنترنت و`WA_BRIDGE_API` |
| QR يظهر ويختفي متكرّراً | الجلسة تالفة — احذف `.session/` وأعد التشغيل |
| `الرقم غير مسجّل على واتساب` | العميل ليس لديه واتساب على هذا الرقم — طبيعي |
| اللوحة تقول «غير متصل» والجسر يعمل | النبضة متأخّرة — انتظر ~10ث |
