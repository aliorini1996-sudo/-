# نظام الـSEO المستمر — FieldSales

دليل مختصر لمنظومة تحسين الظهور في محركات البحث، وكيف تتطوّر **ذاتياً**.

## 1) الـSEO لكل صفحة
- خطّاف `src/lib/seo.ts` (`useSeo`) يضبط لكل صفحة: العنوان، الوصف، الكلمات المفتاحية، `canonical`، Open Graph/Twitter، `og:locale`، و`hreflang`، وبيانات Schema.org.
- مطبّق على: الصفحة الرئيسية، من نحن، تواصل، الشروط/الخصوصية/اتفاقية الخدمة، وصفحتَي المدوّنة.
- `index.html` يحمل البيانات المنظّمة الأساسية: `Organization` + `SoftwareApplication` (مع `featureList` لكل خدمات النظام) + `FAQPage`.

## 2) الدولي (عربي + إنجليزي)
- العربية على الجذر (`/`, `/about`, …) والإنجليزية على بادئة **`/en`** (`/en`, `/en/about`, …).
- اللغة تُشتقّ من المسار (`src/i18n/locale.ts`)؛ زر تبديل اللغة **ينتقل** بين الرابطين (روابط منفصلة قابلة للفهرسة).
- وسوم **`hreflang`** متبادلة (ar ↔ en ↔ x-default) في كل صفحة وفي خريطة الموقع.
- لإضافة لغة جديدة: أضِف ترجمتها في `defaultContentEn`-نمط + مسارات `/<lang>` + بدائل hreflang.

## 3) خريطة الموقع الذكية
- تُولَّد آلياً بـ`scripts/gen-sitemap.mjs`: المسارات (عربي + `/en` مع hreflang) + المدوّنة.
- تجلب **المقالات الحيّة** من الـCMS العام (`/api/site-content`) فتدخل مقالات السوبر أدمن الجديدة تلقائياً؛ وتعود للمقالات الافتراضية عند تعذّر الاتصال.
- تعمل عند **كل بناء** عبر سكربت `prebuild` — فكل نشر يحدّث الخريطة.

## 4) التطوير الذاتي المستمر (الورك فلو)
يعمل على ثلاث طبقات تلقائياً:
1. **عند كل نشر:** `prebuild` يعيد توليد الخريطة من المحتوى الحيّ.
2. **أسبوعياً بنفسه:** GitHub Action `‎.github/workflows/seo-maintenance.yml‎` (كل اثنين) يجلب المقالات الحيّة، يحدّث الخريطة ويلتزمها إن تغيّرت، ثم يدقّق الـSEO وينبّه عند أي تراجع.
3. **يدوياً وقت الحاجة:**
   - `npm run seo:sitemap` — تحديث الخريطة.
   - `npm run seo:audit` — تقرير صحّة SEO كامل.
   - `npm run seo:maintain` — الاثنان معاً.

> ملاحظة: لتفعيل الالتزام التلقائي للـAction، فعّل صلاحية الكتابة في إعدادات المستودع:
> Settings → Actions → General → Workflow permissions → **Read and write permissions**.

## 5) ما يفحصه التدقيق (`seo-audit.mjs`)
العنوان/الوصف وأطوالهما، canonical، OG/Twitter، viewport، الأيقونة، صحّة JSON-LD و`featureList`، robots، صحّة الخريطة و**تبادل hreflang**، تغطية المقالات، واستدعاء `useSeo` في كل صفحة عامّة. يُنهي بخطأ عند أي خلل (لاستخدامه بوّابةً في CI).

## 6) خطوات يدوية في Google Search Console (عند كل تحديث كبير)
- أرسِل `https://fieldsa.net/sitemap.xml`.
- اطلب فهرسة الصفحات الجديدة (مثل `/en` والمقالات).
